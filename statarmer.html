Stat Armer

Stat Armer is python based desktop app designed to centralize and manage site arming during statutory holidays across four monitoring platforms: Immix, Web relays, Chekt, and Ajax. This application ensures consistent holiday scheduling, reduces manual intervention, and enables real-time adjustments based on customer requests.

Data

Stat Armer retrieves data from a SQL database and external APIs. Access to the application is restricted to authorized users within the organization through trusted authentication mechanisms, providing greater control for the IT team and enhancing overall security. A separate server is dedicated to running the arming and disarming scripts. These scripts process entries stored in the Stat Armer database, ensuring a clear separation between application operations and script execution for improved security, reliability, and performance.

User Interface

The user interface is designed with a centralized approach to provide a seamless and intuitive user experience. Customers can access multiple platforms through dedicated buttons, with each button opening a separate page corresponding to the respective platform. The application also allows users to create and manage holiday entries. When a holiday entry is added, it automatically overrides the existing schedule for the specified period, ensuring that operations align with holiday requirements and reducing the need for manual schedule adjustments.

Features

The application provides users with comprehensive manual control over site operations. Users can view schedules, perform arming and disarming actions, and modify entries even after they have been activated. To ensure accountability and maintain a complete audit trail, all user activities are logged and associated with the user's Windows credentials. This enables administrators to track actions, review changes, and investigate operational events when necessary. The audit logging feature enhances security, compliance, and operational transparency across the platform.

Multiple Email

With Version 3.0, users can send bulk email notifications to customers within a specific city, province, or country with a single click. The system utilizes predefined email templates, which can be selected from a dropdown menu to ensure consistent and professional communication. When a mass email campaign is initiated, the application automatically queries the customer database and identifies the primary contact associated with each customer record. Emails are then distributed to these designated contacts, streamlining communication while reducing manual effort. This feature enables efficient outreach for announcements, service updates, maintenance notifications, and other customer communications.

Project History

The development of Stat Armer took more than six months to complete, and it continues to evolve with new features being added regularly. Today, it is one of the most widely used applications within Radius, supporting daily operations and improving efficiency across multiple teams. The project originated from the challenges encountered after introducing products such as AJAX and CHekT to Radius. While these platforms provided scheduling capabilities, they lacked support for statutory holidays and other custom scheduling requirements. Additionally, the process of configuring holiday schedules for Immix was highly manual and time-consuming. It involved multiple steps using Python scripts, Excel spreadsheets, and extensive data processing, often taking up to a week to complete. During every statutory holiday, a team member also needed to remain on call to handle server issues or schedule changes. Stat Armer was developed to address these limitations by centralizing scheduling and automation within a single application. The system stores all scheduling information in a database, allowing users to manage holiday schedules, update entries, and make operational changes directly through the application. Users can also contact customers, create schedules in advance, and make modifications without requiring technical intervention. This significantly reduces administrative effort, improves accuracy, and increases operational efficiency. The initial plan was to build the solution using the Microsoft Power Platform. However, after evaluating the anticipated number of users and the scale of deployment, it became clear that the licensing costs associated with providing Power Apps Pro licenses to all users would not be cost-effective. As a result, the application was developed in Python, providing greater flexibility, scalability, and long-term cost savings while meeting all operational requirements.
